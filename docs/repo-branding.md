# Repo Branding Guide

## Banner Concept

The repository banner should communicate three things immediately:

1. This is healthcare-focused
2. This is about claim validation and policy rules
3. This is both technical and operations-friendly

## Included Banner Asset

Use the default banner included at:



It is suitable for:

- top of README
- LinkedIn project thumbnail reference
- GitHub social preview replacement if you later export it as PNG

## Visual Direction

- Primary color: medical teal
- Secondary color: deep insurance/navy tone
- Accent color: warm amber for review/action
- Tone: clean, trustworthy, modern, operations-friendly

## Banner Headline

`Claim Validation & Policy Rule Engine`

## Banner Subheadline

`Normalize raw claim JSON, apply policy rules, and guide action before submission.`

## Suggested Social Preview Text

`Healthcare Claim Validation Platform`
