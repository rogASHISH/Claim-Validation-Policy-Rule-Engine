# Screenshot Guide

## Recommended Screenshot Files

Save screenshots in this folder using these names:

1. `claim-review.png`
2. `audit-history.png`
3. `policy-rules.png`
4. `reports-dashboard.png`

## What Each Screenshot Should Show

### 1. Claim Review

Show:

- claim upload area
- policy upload area
- verification result
- issues requiring action

### 2. Audit History

Show:

- audit timeline
- selected audit summary
- comparison note with previous run

### 3. Policy Rules

Show:

- built-in rules
- custom policy rules
- inferred field mappings

### 4. Reports Dashboard

Show:

- total audits
- rejection-risk trend
- repeated issue section
- operational report table

## How To Capture

1. Start backend
2. Start frontend
3. Load demo data
4. Run verification
5. Capture each tab after data is visible
